4. Data model (Postgres)

erDiagram
  file_uploads ||--o{ file_upload_rows : upload_id
  file_uploads ||--o{ invites : upload_id
  file_uploads ||--o{ issue_field_edits : upload_id
  file_uploads {
    bigint id PK
    text original_name
    timestamptz created_at
    int row_count
  }
  file_upload_rows {
    bigint id PK
    bigint upload_id FK
    int row_index
    jsonb cells
  }
  invites {
    bigint id PK
    uuid token UK
    bigint upload_id FK
    text invitee_email
    timestamptz expires_at
  }
  issue_field_edits {
    bigint upload_id PK_FK
    text issue_key PK
    text spillover_reason
    text spillover_category
  }

5. Sequence: admin upload

sequenceDiagram
  participant B as Browser analyzer.html
  participant S as Express
  participant D as Postgres
  B->>S: POST /api/upload file
  S->>S: XLSX parse rows
  S->>D: INSERT file_uploads and rows
  B->>S: GET /api/uploads
  S->>D: SELECT uploads
  B->>S: GET /api/uploads/id/data
  S->>D: SELECT rows

6. Sequence: contributor invite

sequenceDiagram
  participant S as Express
  participant M as MailerSend or SMTP
  participant C as Contributor browser
  S->>M: send invite email
  M->>C: Link user-dashboard with token and api base
  C->>S: GET /api/invite/token/session
  S->>C: JSON session
  C->>S: PUT /api/invite/token/issue
  S->>C: OK
