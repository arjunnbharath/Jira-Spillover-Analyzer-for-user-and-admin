2. Main responsibilities

LayerRole
analyzer.html Single-page UI: upload/setup, dashboards, assignees, comments (threads, PATCH/DELETE), invites, ticket drill-down. Uses fetch to /api/... and optional ?api= when the page is not served from the API host.
express-app.js HTTP server: static assets, health, spreadsheet upload, upload listing and data, comments CRUD, issue edits, people insights, invite flows, optional email.
PostgreSQL Persistence for uploads, parsed rows, comments, invites, contributor/issue edits.
Firebase Client-side sign-in; comment author fields come from the signed-in user in the UI.

4. Typical analyst flow (comments + data)

sequenceDiagram
    participant User as Analyst browser
    participant App as analyzer.html
    participant API as express-app.js
    participant PG as PostgreSQL

    User->>App: Open app (same host or ?api=)
    App->>API: GET /api/health (probe)
    App->>API: GET /api/uploads
    API->>PG: list uploads
    PG-->>API: rows
    API-->>App: JSON

    User->>App: Select upload / load workspace
    App->>API: GET /api/uploads/:id/data
    API->>PG: load sheet + edits
    PG-->>App: JSON

    User->>App: Comments tab / refresh
    App->>API: GET /api/uploads/:id/comments
    API->>PG: comments
    PG-->>App: JSON

    User->>App: Send / edit / delete comment
    App->>API: POST or PATCH or DELETE …/comments…
    API->>PG: write
    PG-->>App: ok + body

5. Run / deploy

npm start runs node express-app.js: one process serves static HTML and /api routes. vercel.json may adapt the same app for serverless; configure DATABASE_URL for PostgreSQL.
